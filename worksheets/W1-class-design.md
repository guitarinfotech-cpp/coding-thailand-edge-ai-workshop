## 1. โจทย์ของทีม

**Problem statement** (1-2 ประโยค บอกว่าจะแก้ปัญหาอะไรในชีวิตจริง):

```
[เขียนที่นี่]
เราจะสร้างระบบ Smart Resource Tracker โดยใช้ Webcam ตรวจจับสถานะอุปกรณ์ในห้อง Maker Space
```

## 2. Class Definition

ระบุ class ที่จะใช้ + เกณฑ์ที่ทำให้แต่ละ class แตกต่าง:

| # | ชื่อ Class | คำอธิบาย (3-4 ประโยค) | สิ่งที่ทำให้ต่างจาก class อื่น |
|---|---|---|---|
| 1 | Empty_Shelf | ตรวจจับภาพพื้นที่ว่างบนชั้นวาง ไม่มีอุปกรณ์ใดๆ | พื้นที่สีพื้นหลังว่างเปล่า ไม่มีขอบวัตถุ | 
| 2 | Tool_Present | ตรวจจับภาพที่มีอุปกรณ์ (เช่น กรรไกร/คีม) วางอยู่ | มีรูปร่างและขอบเขตวัตถุที่ชัดเจน |
| 3 | Hand_Reaching | ตรวจจับภาพที่มีมือนักเรียนกำลังเอื้อมมาหยิบหรือคืนของ | มีรูปร่างนิ้วมือหรือผิวหนังมนุษย์ในเฟรม |

**ตัวอย่าง (Track A, Fall Detector):**

| # | ชื่อ Class | คำอธิบาย | สิ่งที่ทำให้ต่าง |
|---|---|---|---|
| 1 | ยืน | ผู้ใช้ยืนนิ่ง หรือยืนเฉยๆ ใส่ที่เอว | accel แกน Z สูง, gyro ต่ำ |
| 2 | เดิน | เดินปกติ ก้าวเท้าสลับ | accel oscillation, periodic |
| 3 | ล้ม | ล้มลงพื้นกะทันหัน | high accel spike + horizontal Z |

## 3. Edge Cases — สิ่งที่อาจสับสน

ระบุสถานการณ์ที่อาจทำให้โมเดลสับสน:

```
1. "วัตถุที่มีสีคล้ายชั้นวาง" อาจทำให้โมเดลมองว่าชั้นวางว่าง (Empty_Shelf)

2. "เงาของมือ" ที่ทอดลงบนชั้นวาง อาจถูกเข้าใจผิดว่าเป็น Tool_Present

3. "การหยิบของเร็วเกินไป" ทำให้กล้องจับภาพไม่ทันหรือภาพเบลอ
```

**วางแผนแก้:**
```
จัดแสงสว่างบริเวณชั้นวางให้สม่ำเสมอ (สว่างคงที่) และใช้ฐานวางอุปกรณ์ที่มีสีตัดกับวัตถุ (Contrast) เพื่อช่วยให้ AI แยกแยะขอบวัตถุได้ชัดเจนขึ้น
```

## 4. Data Collection Plan

| Class | จำนวน samples | ระยะเวลา/sample | สภาพแวดล้อมที่จะเก็บ |
|---|---|---|---|
| Empty_Shelf | 100 images | - | ชั้นวางใน Maker Space แสงปกติ |
| Tool_Present | 100 images | - | ชั้นวางที่มีอุปกรณ์วางอยู่จริง |
| Hand_Reaching | 100 images | - | มุมกล้องเดียวกับการใช้งานจริง |

**Total samples ทั้งหมด:** 300 images
**คาดว่าใช้เวลาเก็บ:** 30 นาที

⚠️ **เป้าหมายขั้นต่ำ:**
- Motion/Audio: 50 samples × class
- Vision: 100 images × class
- Environment: 200 samples × class

## 5. Diversity Plan — กันอคติ (Bias)

ตอบ 3 คำถาม:

### a) มีหลาย "environment" ไหม?
```
เก็บภาพในระดับแสงที่ต่างกัน (เปิดไฟห้องเรียน/ปิดม่าน) และเปลี่ยนตำแหน่งกล้องเล็กน้อยเพื่อความทนทาน
```

### b) มีหลาย "subject" ไหม? (กรณี Vision/Audio/Motion)
```
ให้เพื่อนหลายๆ คนในทีมสลับกันเอื้อมมือมาหยิบของ เพื่อให้ AI จำมือที่ขนาดต่างกันได้
```

### c) มีหลาย "variation" ไหม?
```
วางอุปกรณ์ในหลายๆ ตำแหน่งบนชั้นวาง และใช้มือหยิบจากหลายทิศทาง
```

## 6. Output Plan

ทีมจะใช้ Modulino อะไรเป็น output? บอกผลลัพธ์ยังไง?

```
[ ] Modulino Pixels — แสดงสีตามคลาส (สีอะไร?)
[ X ] Modulino Buzzer — เสียง alert (ทำเมื่อ?)
[ ] Modulino LED Matrix — แสดง pattern
[ ] อื่นๆ: ________________________________
```

- Empty_Shelf → ไฟ RGB (ถ้ามี) สีส้ม (แจ้งสถานะว่าง)
- Tool_Present → ไฟ RGB สีเขียว (พร้อมใช้งาน)
- Hand_Reaching → Buzzer ดัง 1 ครั้ง (เพื่อยืนยันการทำรายการ) + LCD แสดง "Processing..."
